# 任务 2：补 App Server Bridge API 测试

新增测试文件：

```text
tests/test_app_server_bridge.py
```

测试目标文件：

```text
poc/app_server/app_server_bridge.py
```

不要启动真实 `codex app-server`。

可以通过 monkeypatch / fake client 的方式替代真实 `JsonlRpcClient`。

至少覆盖：

1. `/health`

   * 不需要 token。
   * 返回 status、service、threads、token_required、idle_timeout_seconds、codex_command、repo_root、mode、sandbox。

2. token 校验

   * 设置 `APP_SERVER_BRIDGE_TOKEN=secret` 后：

     * `/health` 仍可访问。
     * `/threads` 无 token 返回 401。
     * `/threads` token 正确返回 200。

3. `POST /threads`

   * 支持 JSON body `{ "title": "xxx" }`。
   * title 为空时使用默认标题。
   * 返回 bridge_thread_id、app_thread_id、title、status。

4. `GET /threads`

   * 返回当前内存中的 threads。
   * 包含 title、status、turn_count、created_at、updated_at。

5. `PATCH /threads/{id}`

   * 能修改 title。
   * 空 title 返回 400。
   * 不存在 thread 返回 404。

6. `DELETE /threads/{id}`

   * 正常删除返回 deleted/closed 类结果。
   * 不存在返回 404。
   * running 或锁被占用时返回 409。
   * 删除时会调用 fake client close。

7. `POST /threads/{id}/turns`

   * 空 message 返回 400。
   * 不存在 thread 返回 404。
   * 同 thread 并发锁被占用返回 409。
   * 成功时写入 turn 产物目录或通过 monkeypatch 隔离到 tmp_path。

8. `GET /threads/{id}/final`

   * 能返回最近一轮 assistant_final。
   * 不存在 thread 返回 404。

测试要求：

* 不启动真实 subprocess。
* 不依赖真实 Codex CLI。
* 使用 tmp_path 隔离 bridge-runs。
* 测试完成后不污染仓库目录。
* 如果当前 bridge 文件不方便测试，可以做最小必要重构，但不要扩大范围。

# 任务 4：修复 backend/mobile.py XSS 风险

当前 `backend/mobile.py` 使用前端模板字符串和 `innerHTML` 渲染动态数据。需要补最小 escape。

修改文件：

```text
backend/mobile.py
```

要求：

1. 在 mobile 页面 JS 中增加：

```javascript
function escapeHtml(value) {
  return String(value ?? "")
    .replaceAll("&", "&amp;")
    .replaceAll("<", "&lt;")
    .replaceAll(">", "&gt;")
    .replaceAll('"', "&quot;")
    .replaceAll("'", "&#39;");
}
```

2. 所有进入 `innerHTML` 的动态字段必须 escape，包括但不限于：

```text
project.name
runner.runner_id
runner.status
runner.hostname
runner.supported_models
task.status
task.task_type
task.assigned_runner_id
task.runner_id
task.model
task.reasoning_effort
task.sandbox
task.log_url
task.result_url
task.diff_url
```

3. 对于 URL 字段，如果是后端返回的相对路径，也要 escape 后再进入 href。
4. 不要大改页面结构。
5. 不要引入前端框架。

补测试：

在现有：

```text
tests/test_ui.py
```

或新增：

```text
tests/test_mobile_ui.py
```

覆盖：

1. 构造包含恶意字符串的 project / runner / task，例如：

```text
<script>alert(1)</script>
```

2. 断言 mobile 页面渲染时不会直接出现未转义的 `<script>`。
3. 断言出现的是转义后的 `&lt;script&gt;` 或通过 JS escapeHtml 逻辑保护。

如果用 TestClient 检查 `/mobile` 页面，只需验证页面中包含 `escapeHtml` 函数，以及关键渲染位置使用了 `escapeHtml(...)`。
