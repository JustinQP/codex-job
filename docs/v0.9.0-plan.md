# 二、总体目标

新增异步 App Turn 链路：

```text
POST /app-threads/{id}/turns/async
  ↓
立即创建 AppTurn(PENDING/RUNNING)
  ↓
返回 AppTurnRead
  ↓

  ↓

  ↓

```

保留原同步链路：

```text

```

# 三、任务 1：新增异步执行器

新增文件：

```text
backend/services/app_turn_executor.py
```

## 职责

负责单进程内的异步 App Turn 后台执行。

使用标准库：

```python
concurrent.futures.ThreadPoolExecutor
threading
```

不要引入 Celery / Redis / APScheduler。

## 要求

1. 创建全局轻量线程池：

```python
ThreadPoolExecutor(max_workers=2)
```

2. 支持环境变量：

```text
APP_TURN_EXECUTOR_MAX_WORKERS
```

默认 `2`。

3. 提供函数：

```python
submit_app_turn(app_turn_id: int) -> None
```

4. 后台任务内部重新创建 DB session，不能复用请求线程传入的 session。

5. 后台任务流程：

```text
读取 AppTurn
读取 AppThread
如果 AppTurn 不存在：直接结束
如果 AppTurn.status 不是 PENDING/RUNNING：直接结束
如果 AppThread 不存在：AppTurn FAILED
如果 AppThread CLOSED：AppTurn FAILED
调用 Bridge send_turn
获取 full final
获取 events summary
更新 AppTurn SUCCESS / FAILED
更新 AppThread ACTIVE / ERROR
```

6. 后台任务异常必须捕获，不允许让线程异常静默丢失。

7. 失败时记录：

```text
AppTurn.status=FAILED
AppTurn.error_message
AppTurn.completed_at
AppThread.status=ERROR
AppThread.last_error
AppThread.updated_at
```

8. 成功时记录：

```text
AppTurn.status=SUCCESS
AppTurn.assistant_final
AppTurn.bridge_turn_id
AppTurn.event_summary_json
AppTurn.completed_at
AppThread.status=ACTIVE
AppThread.last_error=None
AppThread.updated_at
```

# 四、任务 2：AppTurn 状态与 Service 增强

修改文件：

```text
backend/services/app_thread_service.py
backend/schemas.py
```

## 状态

当前已有：

```text
RUNNING
SUCCESS
FAILED
```

本次增加：

```text
PENDING
```

不要改成 enum，继续使用 str。

## 新增 Service 函数

在 `app_thread_service.py` 中新增：

```python
create_async_app_turn(session, app_thread_id, payload) -> AppTurn
get_app_turn_or_404(session, app_turn_id) -> AppTurn
```

### create_async_app_turn 行为

1. 校验 message 非空。
2. 校验 AppThread 存在。
3. 如果 AppThread.status == CLOSED，返回 400。
4. 如果 AppThread 没有 bridge_thread_id，返回 400。
5. 创建 AppTurn：

```text
status=PENDING
created_at=now
started_at=None
completed_at=None
```

6. commit 后调用：

```python
submit_app_turn(app_turn.id)
```

7. 返回 AppTurn。

### get_app_turn_or_404 行为

1. 查找 AppTurn。
2. 不存在返回 404。
3. 返回 AppTurn。

## AppTurnRead 增强

修改 `AppTurnRead`，增加：

```text
duration_seconds: Optional[float] = None
```

计算逻辑：

```text
如果 started_at 和 completed_at 都有，则返回两者秒差。
否则 None。
```

不新增数据库字段。

点击“异步发送 App Turn”：

1. 校验已选择 AppThread。
2. 校验 message 非空。
3. 如果 AppThread CLOSED，提示先重开。



```text

```

```text

```

# 七、任务 5：smoke 脚本支持异步模式

修改文件：

```text
scripts/smoke_app_server_flow.py
```

新增参数：

```text
--async-turn
--poll-interval-seconds 默认 2
--poll-timeout-seconds 默认 300
```

## 同步模式

保持当前行为不变。

## 异步模式行为

如果传入 `--async-turn`：

1. 创建 AppThread 后调用：

```text
POST /app-threads/{id}/turns/async
```

2. 获取 turn_id。
3. 循环调用：

```text
GET /app-turns/{turn_id}
```

4. 每次等待 `poll_interval_seconds`。
5. 超过 `poll_timeout_seconds` 仍未 SUCCESS/FAILED，则失败。
6. SUCCESS 后继续调用：

```text
GET /app-threads/{id}/final
GET /app-threads/{id}/turns
GET /app-threads/{id}/events
DELETE /app-threads/{id}
```

7. FAILED 则输出 error_message，尽量 cleanup，exit code 1。

## 输出增加

summary 增加：

```text
async_turn: true/false
poll_count
final_turn_status
```

## 清理

沿用 v0.8.3 的 cleanup 机制。无论同步或异步，只要 AppThread 已创建，失败时尽量 DELETE。
