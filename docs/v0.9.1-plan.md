请基于当前 `JustinQP/codex-job` 项目，完成 `v0.9.1：异步 App Turn 并发保护与取消/超时体验`。

## 背景

当前项目已完成：

```text
v0.8.0：App Server 会话模式主线集成
v0.8.1：完整 final 保存、bridge_thread_id 校验
v0.8.2：主线 App Server smoke 脚本、mobile 操作体验增强
v0.8.3：一键启动、smoke 失败清理、AppThread reopen 恢复能力
v0.9.0：App Turn 异步执行与手机端轮询
```

当前异步链路：

```text
POST /app-threads/{id}/turns/async
  ↓
创建 AppTurn(PENDING)
  ↓
ThreadPoolExecutor 后台执行
  ↓
GET /app-turns/{id} 轮询状态
```

当前问题：

1. 同一个 AppThread 可以同时创建多个 `PENDING/RUNNING` AppTurn。
2. 用户在 mobile 连续点击“异步发送”可能导致 Bridge thread 并发冲突或上下文顺序混乱。
3. 异步 turn 没有本地取消能力。
4. 异步 turn 没有明确超时标记能力。

本次目标是增强异步 App Turn 的受控性，不做大功能。

修改文件：

```text
backend/services/app_thread_service.py
backend/schemas.py
```

当前 AppTurn 状态有：

```text
PENDING
RUNNING
SUCCESS
FAILED
```

新增：

```text
CANCELLED
```

继续使用 str，不引入 enum。

取消语义：

```text
如果 AppTurn 还没开始执行：可以标记 CANCELLED。
如果 AppTurn 已 RUNNING：只做本地取消标记，不保证中断 Bridge/Codex。
后台 executor 在关键步骤检查状态，发现 CANCELLED 后尽量停止后续更新。
```

# 四、任务 3：新增取消 AppTurn API

修改文件：

```text
backend/main.py
backend/services/app_thread_service.py
backend/schemas.py
```

新增 service 函数：

```python
cancel_app_turn(session, app_turn_id) -> AppTurn
```

行为：

1. AppTurn 不存在返回 404。
2. 如果 AppTurn.status in `SUCCESS/FAILED/CANCELLED`，直接返回当前 AppTurn，不报错。
3. 如果 AppTurn.status in `PENDING/RUNNING`：

   * 设置 `status=CANCELLED`
   * 设置 `error_message="cancelled by user"`
   * 如果 `started_at is None`，保持 None。
   * 设置 `completed_at=now`
   * commit
4. 如果该 AppTurn 是其 AppThread 最新活动 turn，可将 AppThread.status 设置为 `ACTIVE`，并清空 `last_error`，不要把 AppThread 留在 ERROR。
5. 不尝试直接中断 Bridge/Codex 子进程。
6. 在注释中明确：当前取消是本地状态取消，不保证中断正在执行的 Codex turn。

新增 API：

```text
POST /app-turns/{app_turn_id}/cancel
```

要求：

1. 需要 `X-API-Token`。
2. 返回 AppTurnRead。
3. 不存在返回 404。

# 五、任务 4：executor 支持取消和超时

修改文件：

```text
backend/services/app_turn_executor.py
```

## 1. 执行前检查 CANCELLED

在 `_execute_with_session()` 中：

1. 读取 AppTurn 后，如果 status 是 `CANCELLED`，直接 return。
2. 设置 RUNNING 前再次确认不是 CANCELLED。

## 2. Bridge 调用后检查 CANCELLED

Bridge `send_turn()` 返回后，在写 SUCCESS 前重新从 DB refresh AppTurn：

```text
如果此时 status == CANCELLED：
  不要覆盖为 SUCCESS
  不要覆盖 assistant_final
  return
```

## 3. 支持本地执行超时配置

新增环境变量：

```text
APP_TURN_EXECUTION_TIMEOUT_SECONDS
```

默认：

```text
600
```

注意：标准库线程无法安全强杀正在阻塞的 Bridge HTTP 请求，但可以把这个值传给 Bridge client timeout。

当前 `AppServerBridgeClient` 默认读 `APP_SERVER_BRIDGE_TIMEOUT_SECONDS`。本次可以在 executor 中：

```text
优先读取 APP_TURN_EXECUTION_TIMEOUT_SECONDS
临时创建 AppServerBridgeClient(timeout_seconds=...)
```

不要影响同步接口。

要求：

1. `APP_TURN_EXECUTION_TIMEOUT_SECONDS` 非法时回退 600。
2. timeout 发生时，AppTurn 标记 FAILED，error_message 包含 timeout。
3. README 写清楚：当前 timeout 是 HTTP 调用超时，不是强杀 Codex 子进程。

# 七、任务 6：smoke 增加并发保护验证

修改文件：

```text
scripts/smoke_app_server_flow.py
```

新增可选参数：

```text
--check-async-conflict
```

行为：

1. 仅在 `--async-turn` 模式下生效。
2. 创建 AppThread 后，先提交第一个 async turn。
3. 在第一个 turn 还未完成时，立即再提交第二个 async turn。
4. 预期第二个请求返回 409。
5. 如果没有返回 409，则 smoke 失败。
6. 然后继续轮询第一个 turn。
7. summary 增加：

```text
async_conflict_checked: true/false
async_conflict_passed: true/false
```

测试中用 fake request 覆盖即可，不要求真实 Bridge 慢响应。
