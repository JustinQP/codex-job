修改文件：

```text id="z2zvkk"

README.md
```

新增 service 函数：

```python id="rv15tf"

```

扫描所有：

```text id="c3nbn9"
AppTurn.status in {"PENDING", "RUNNING"}
```

将其标记为：

```text id="rzey6h"
status = FAILED

```

对应 AppThread：

```text id="s2n2v8"

updated_at = now
```

返回：

```json id="e48m80"
{
  "recovered_count": 2,
  "recovered_turn_ids": [1, 2]
}
```

要求：

1. 已经是 SUCCESS / FAILED / CANCELLED 的 AppTurn 不处理。
2. 不删除任何历史数据。
3. 不自动重放用户消息。
4. 不尝试恢复后台线程。
5. 这是“失败恢复”，不是“继续执行”。

## 3.3 增加关闭全部 CLOSED/ERROR 清理能力

不要物理删除数据库记录。

新增 API：

```text id="o5dodl"
POST /app-threads/cleanup
```

请求：

```json id="pyf0og"
{
  "status": "CLOSED",
  "limit": 50
}
```

行为：

1. 只支持清理 `CLOSED` 或 `ERROR` 的 AppThread。
2. 清理不是删除，而是标记 title 前缀：


4. 返回：

```json id="dnz8rl"
{
  "archived_count": 3,
  "archived_thread_ids": [1,2,3]
}
```

目的：

```text id="x5fb8m"
让列表可以筛掉 archived，减少手机页面干扰。
```
